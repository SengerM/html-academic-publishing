Introduction

One key point in science is to share your results with other people. The most usual way to do this is by producing a PDF document that contains just what a PDF file can contain: Text and images. But what if you want to share a video? What if you want to share something that is interactive? What if you want to read it in your mobile? Or increase the font size? You can't do this with a PDF document (or you'll have a hard time in doing it). Thus, though the PDF format is practically unbeatable in accomplishing its design objective (i.e. to produce a digital version of a “paper document”), it's a pity to limit the capability of documents to what this format allows to do. After all, what is the fraction of PDF files that you actually print? I think we need a change in paradigm here: To abandon some concepts tied to paper-printed documents and start producing material for monitor-rendered documents.

What can replace the PDF format?

PDF is nowadays the most extended format for sharing documents. If you receive a PDF file, it does not matter which device you use or which operating system or which software, you will find a way to read it. If you are in a mobile device you will probably suffer of reading a PDF in such a small screen, but you will definetly be able to read it. So if we are going to replace the PDF format, we need something that is at least as universal as the PDF. We also need something that can handle all the stuff that a PDF file can contain, this however is not complicated as this consists just in images and text.

So, which is a good alternative to replace PDF? Well, if you think of websites they can show text, images, and they render good with appropriate text size either in a computer or in a mobile. It does not matter which operating system do you use or which web browser, the web site will display fine. This looks promising.

The progress that has been done in the last decades with HTML is amazing. You can show literally any type of content and display it however you want. So the technologies behind web sites look as a good option to replace the PDF format. Furthermore, these technologies are free, cross platform (Windows, Mac, Linux, Android, etc), device friendly (computer, cell phone, tablet, etc), easy share, etc.

Still want a PDF?

You still want a PDF? Open this document with your favourite web broser, hit ctrl+print and save it as a PDF file. Firefox can do this, probably all other "famous" browsers can too. You may have to set up the zoom, margins, background, etc. such that it looks fine as in the browser. But believe me, it is trivial to get a good looking PDF from this.

Latex vs HTML

In physics, math and other related fields of science it is very common to write documents using Latex. Latex is very nice and powerful, it produces amazing PDF files. But this is preciselly the problem I see: It produces PDFs. If you write in Latex it is because you want a PDF.

HTML, as Latex, is a markup language but, in contrast, it is much more widespread due to its masive usage in websites. The comunity of HTML users is infinitely bigger than the comunity of Latex users and HTML is beautifully rendered by any modern web browser available in any device.

So, suppose you want to embed a video in your document, or produce a document which adapts itself to the width of the window in which your are reading it, or that has some interactive feature. You can probably implement this for Latex, but you will have a lot of workI guess you will have to invent some new format from scratch.. For HTML this is already done! You just have to write your document and open it with a web browser. But the machinery that makes it work is already implemented and spread around the world.

In view of this it looks very reasonable to choose HTML as the tool for "the next generation of scientific documents" rather than Latex or anything else.

Sections and sub-sections

Mathematic equations

Somebody has already implemented this for HTML. Fortunately the support for mathematic equations can be added just by linking a script with a few lines in the header of the HTML file. Then you can just write your equations using Latex code, which is very nice for equations. You can add inline equations $x^2+x+1=0$ and you can add "display" equations $$\intop_0^1f(x)dx=\pi\text{.}$$ The previous equation is numbered. What happens if I insert an equation with no id? $$E=mc^2$$ The equation is numbered, though I cannot insert a reference to it because it has no id. Now I have another important equation $$\left( i \gamma^\mu \partial_\mu - m \right) \psi = 0$$ that later on I will cite. Now I will try a very long equation: $$\frac{dF}{dx}\frac{dx}{dy}\frac{dy}{d\xi}=\intop_{\mathbb{R}}f(\zeta)e^{-\zeta}\cos(w^{2}-\zeta)\ d\zeta+\frac{3x^{3}+(\nabla\times\boldsymbol{E})\cdot\boldsymbol{B}}{\vec{r}\cdot\vec{E}}+8\pi\hbar+\log\cos\tan\sin\pi$$

Is it possible to define our custom Latex commands? This would be extremly useful. So I have this: $$f(x) \overset{\text{def}}{=} \frac{\sin x}{x}$$ where the $\overset{\text{def}}{=}$ is hardcoded. Can I define the command \DEF that implements this symbol? So here I define the symbol: $\def\DEF{\overset{\text{def}}{=}}$ and now I will try to use it: $$f(x)\DEF\frac{\cos x}{x}$$ Yes!! It worked!
