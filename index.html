Introduction

One key point in science is to share your results with other people. The most usual way to do this is by producing a PDF document with either slides or “paper pages” that contains just what a PDF file can contain: Text and images. But what if you want to share a video? What if you want to share something that is interactive? You can't do this with a PDF document. Nowadays filming videos and producing interactive media has become very accessible to everyone. Thus, though the PDF format is practically unbeatable in accomplishing its design objective (i.e. to produce a digital version of a “paper document”), it's a pity to be stuck with the PDF format since the 90s. I think we need a change in paradigm here: To abandon some concepts tied to paper-printed documents and start producing material for monitor-rendered documents. After all, what is the fraction of PDF files that you actually print?

What can replace the PDF format?

PDF is nowadays the most extended format for sharing documents. If you receive a PDF file, it does not matter which device you use or which operating system or which software, you will find a way to read it. So if we are going to replace the PDF format, we need something that is at least as universal as the PDF.

But how can we do this? Well, if you think of websites the progress that has been done in the last decades is amazing. You can show literally any type of content and display it however you want. So the technologies behind web sites look promising to replace old school PDF files. Furthermore, these technologies are free, cross platform (Windows, Mac, Linux, Android, etc), device friendly (computer, cell phone, tablet, etc), easy to share, and if you still want to have a PDF file you can just ctrl+p the document and produce a nice PDF separated in "paper pages". One example of such a "scientific HTML publication" is this document that you are reading.

Latex vs HTML

In physics, math and other related fields of science it is very common to write documents using Latex. Latex is very nice and powerful, it produces amazing PDF files. Latex is a markup language, as it is HTML. The problem with Latex is that the community of users it has, as compared to HTML, is very small. Furthermore the documentation for Latex is very poor, if existent at all. In contrast, HTML is very well documented and the community of users is infinitely bigger. Also, web browsers cannot render Latex files. So if you write a document in Latex your (almost) only option is to produce a PDF file to share. Thus, although Latex is very popular in our community and it is a very powerful tool, I think it is time to replace it by something bigger as HTML. If you are used to writing in Latex, switching to this type of HTML document is very easy.

Sections and sub-sections

Scientific documents are structured in sections, sub-sections, sub-sub-sections and so on. To accomplish this in HTML you can use the <h1>, <h2>,... tags. The scripting engine I have linked to this HTML will automatically number the sections and also produce a table of contents.
